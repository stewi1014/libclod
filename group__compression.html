◆ clod_compression_algo

enum clod_compression_algo

Compression algorithms. Support for compression libraries and hence algorithms can be configured at compile time and can be checked with clod_compression_support.

Currently, the library compiles with support for all compression algorithms by default.

Enumerator
CLOD_UNCOMPRESSED 

No compression. Provided by memcpy lol. Uncompressed size is always known.

CLOD_GZIP 

GZip compression, provided by libdeflate. Uncompressed size is not known on failure.

CLOD_ZLIB 

ZLib compression, provided by libdeflate. Uncompressed size is not known on failure.

CLOD_DEFLATE 

Deflate compression, provided by libdeflate. Uncompressed size is not known on failure.

CLOD_LZ4 

LZ4 compression, provided by liblz4. Uncompressed size might be known on failure if not malformed.

CLOD_LZMA 

LZMA compression, provided by liblzma.

CLOD_ZSTD 

ZSTD compression, provided by libzstd. Uncompressed size is known on failure if not malformed.

CLOD_MINECRAFT_LZ4 

Custom minecraft LZ4 variant. Some very unfortunate design decisions must have happened to land us with this. Uncompressed size is known on failure if not malformed.

◆ clod_compression_level

enum clod_compression_level

The compression level to use. These are mapped to each compression algorithm's useful range.

Enumerator
CLOD_COMPRESSION_LOWEST 

Lowest compression level. Fastest with the largest compressed sizes.

CLOD_COMPRESSION_LOW 

Low compression level. Faster with larger compressed sizes.

CLOD_COMPRESSION_NORMAL 

Normal compression level. Uses uses algorithm default values.

CLOD_COMPRESSION_HIGH 

High compression level. Slower with smaller compressed sizes.

CLOD_COMPRESSION_HIGHEST 

Highest compression level. Slowest with the smallest compressed sizes.

◆ clod_decompress()

enum clod_compression_result clod_decompress ( struct clod_decompressor * ctx,
void * dst,
size_t dst_size,
const void * src,
size_t src_size,
size_t * actual_size,
enum clod_compression_algo algo )

Decompress some data with the given algorithm.

It is recommended to always know the decompressed size ahead of time, and as such, use a null actual_size value. If actual_size is null and the decompressed output is smaller than dst_size, the method will fail and return CLOD_COMPRESSION_SHORT_OUTPUT. If dst_size is smaller than the decompressed output, the method fails with CLOD_COMPRESSION_SHORT_BUFFER. Sometimes compressed data formats store an uncompressed size in a header, which allows the uncompressed size to be known without decompressing successfully. In that case, actual_size is set to the uncompressed size even on failure.

Parameters
[in]ctxDecompressor.
[out]dstWhere decompressed data is written.
[in]dst_sizeSize of dst.
[in]srcWhere compressed data is read from.
[in]src_sizeSize of compressed data in src.
[out]actual_sizeThe actual size of the decompressed output.
algoThe compression algorithm to use.
Returns
The result of the decompression.
Exceptions
CLOD_COMPRESSION_SUCCESSOn successful decompression.
CLOD_COMPRESSION_UNSUPPORTEDIf the compression algorithm is unsupported.
CLOD_COMPRESSION_MALFORMEDIf the compressed data is malformed. THIS IS NOT AN INTEGRITY CHECK.
CLOD_COMPRESSION_SHORT_BUFFERIf dst is too small to hold the decompressed output.
CLOD_COMPRESSION_SHORT_OUTPUTIf dst is larger than the decompressed output and bytes_written is null.
CLOD_COMPRESSION_ALLOC_FAILEDIf memory allocation failed.
