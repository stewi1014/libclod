uint64_t(* clod_table_opts::hash_func) (uint64_t seed, const void *key, size_t key_size)

Custom hash function. This map implementation demands a uniform spread of entropy across all bits in uint64, and that two keys with nonequal hashes are also not equal when using the cmp_func method.

◆ cmp_func

int(* clod_table_opts::cmp_func) (const void *key1, const void *key2, size_t key_size)

Custom equality function. The required behaviour for the hash_func and cmp_func relationship is; assert(cmp_func(a, b) != 0 || hash_func(a) == hash_func(b)); In other words, if cmp_func thinks two elements are equal, hash_func must agree.
